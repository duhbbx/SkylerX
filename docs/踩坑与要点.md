# 踩坑与要点

本项目的踩坑与解法增量沉淀。只记要点，按主题归类，方便速查。

## 达梦 DM

- **对象类型在 `all_objects` 里是 `object_type='CLASS'`，不是 Oracle 的 `'TYPE'`。** 列对象类型、查类型 DDL 都要用 `CLASS`：列表 `WHERE object_type='CLASS'`；`dbms_metadata.get_ddl('CLASS', …)`（用 `'TYPE'` 报 -20008），类型体是 `'CLASS_BODY'`（带空格的 `'CLASS BODY'` 报 -20001）。其余对象类型（TABLE/VIEW/FUNCTION/PROCEDURE/PACKAGE/SEQUENCE/TRIGGER/SYNONYM）与 Oracle 一致。见 `core-driver/dialects/dm.ts`、`ui/ddl.ts`。
- **`all_tables.dropped` 在 DM 是 `NULL`（无回收站），不是 Oracle 的 `'YES'/'NO'`。** 过滤回收站对象要用 `dropped IS NULL OR dropped <> 'YES'`，直接 `dropped='NO'` 会把 DM 的表全滤光。见 `ui/migrate/introspect.ts` 的 `readOracle`（Oracle/DM 共用）。

## SQL 执行

- **语句拆分器必须识别 PL/SQL 块，不能按裸分号切。** `BEGIN … NULL; END;` 里的分号是块内分隔，按 `;` 拆会把触发器/函数/过程/包体从中间切断（达梦/Oracle 报 -2007 语法错）。`ui/sqlSplit.ts` 的规则：`CREATE [OR REPLACE] TRIGGER/FUNCTION/PROCEDURE/PACKAGE/TYPE`、`DECLARE`、匿名 `BEGIN` 块内的 `;` 不拆，由单独一行的 `/`（SQL*Plus 风格）或输入结束终止；`BEGIN;`/`BEGIN WORK` 等事务控制不算 PL/SQL；PG 的 `$$ … $$` 函数体里的 `;` 也要保护。

## 跨方言 introspection（reader）

- **`information_schema.COLUMNS` / `all_tab_columns` 不区分基表与视图。** 直接按列建表会把视图也当成表。各 reader 要先取基表名再按基表过滤：MySQL/SQLServer 用 `TABLE_TYPE='BASE TABLE'`，Oracle/DM 用 `all_tables`。只有 PG reader 因为用 `relkind IN ('r','p')` 天生免疫。
- **MySQL 约束名只在表内唯一**（每个表主键都叫 `PRIMARY`，FK 名也可跨表重名）。按约束聚合的 Map 要用 `表+约束名` 复合键，否则两表同名约束会串列（如主键变成 `['id','id']`）。

## 主题 / Monaco

- **主题支持「跟随系统」**：`settings.theme` 现为 `'dark'|'light'|'system'`，默认 `system`（老用户保留已存值）。`settings.ts` 用 `window.matchMedia('(prefers-color-scheme: dark)')` 解析，并监听 OS 切换；导出响应式 `resolvedTheme`（解析后的 dark/light）。`data-theme` 属性据此切换 CSS 变量。
- **Monaco 编辑器配色要跟随主题**：之前 `SqlEditor.vue` 硬编码 `theme:'vs-dark'`，浅色模式下编辑器还是黑的。改为 watch `resolvedTheme` 调 `monaco.editor.setTheme('vs'|'vs-dark')`（全局 API，一次切全部实例）。全项目只有 `SqlEditor.vue` 调 `monaco.editor.create`。
